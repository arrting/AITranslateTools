# Kindle 自動翻譯筆記工作流 — 使用說明

適用腳本：`fixed_region_watcher.py`（固定範圍自動偵測翻頁版）+ `find_coordinates.py`（座標校準工具）

---

## 一、一次性設定（只要做一次）

### 步驟 1：安裝 Python 套件

開啟終端機（PowerShell 或 CMD），執行：

```
pip install pyautogui winocr pillow google-genai win11toast
```

### 步驟 2：申請 Gemini API Key

1. 前往 [Google AI Studio](https://aistudio.google.com/) 建立 API Key
2. **建議用個人電腦、個人 Gmail 帳號設定**，避免公司/機構的瀏覽器政策擋掉申請流程

### 步驟 3：設定環境變數

Windows（PowerShell）：

```
setx GEMINI_API_KEY "你的API金鑰"
```

設定完後**關掉終端機重開**，環境變數才會生效。

### 步驟 4：把兩支腳本放到同一個資料夾

例如 `find_coordinates.py`、`fixed_region_watcher.py` 放在同一個你自己選的資料夾裡。

### 步驟 5：修改 `fixed_region_watcher.py` 設定區

打開檔案，找到最上面的設定區，把這幾行換成你自己的：

```python
NOTE_FILE = r"C:\Users\你的名字\Documents\ObsidianVault\Kindle小說筆記.txt"
TRANSLATION_FILE = r"C:\Users\你的名字\Documents\ObsidianVault\Kindle小說筆記_翻譯.txt"
BOOK_TITLE = "你正在讀的小說書名"
CHAPTER = "第X章"
```

`REGION` 這一行先跳過，下一步會用工具幫你找出正確數值。

### 步驟 6：校準座標

1. 把 Kindle 視窗開好，調整到你**平常閱讀習慣的大小和位置**（之後盡量固定，不要常常變動視窗大小）
2. 執行：

```
python find_coordinates.py
```

3. 依照終端機提示，5 秒內把滑鼠移到文字內容區塊的**左上角**，再移到**右下角**
4. 程式會印出類似這樣的結果：

```
REGION = (120, 180, 1150, 850)
```

5. 把這行複製，貼到 `fixed_region_watcher.py` 的 `REGION = (...)` 那一行，取代原本的數值

一次性設定到此完成。

---

## 二、每次閱讀操作（之後每次讀書只要做這幾步）

1. 打開終端機，切到腳本所在資料夾
2. 執行：

```
python fixed_region_watcher.py
```

3. 看到「📖 自動偵測翻頁模式已啟動」就代表準備好了
4. 打開 Kindle，開始正常閱讀、翻頁
5. 每翻一頁，程式會自動截圖、辨識、翻譯，完成後螢幕右下角會跳出通知，同時筆記檔案會自動更新
6. 讀完想結束時，回到終端機視窗按 `Ctrl + C`

---

## 三、跨 session 閱讀（讀到一半關掉，之後接著讀）

- 筆記是依照 `CHAPTER` 這個變數分章節存放的，每次開始新的閱讀 session 前，記得先確認/更新 `fixed_region_watcher.py` 裡的 `CHAPTER` 是不是你目前讀到的章節
- 如果你往回翻閱補看漏掉的內容，程式會自動把新內容插入到對應章節的區塊裡，不會因為操作的先後順序而把筆記順序弄亂

---

## 四、常見問題排查

| 狀況 | 可能原因 | 解決方式 |
|---|---|---|
| 程式完全沒反應，翻頁也沒觸發翻譯 | `REGION` 座標不準，截到的範圍沒有文字或位置跑掉 | 重新執行 `find_coordinates.py` 校準，Kindle 視窗位置/大小不要中途改變 |
| 一直誤判成「翻頁了」（畫面沒變卻觸發翻譯） | `CHANGE_THRESHOLD` 太敏感 | 把數值調高，例如從 `5` 改成 `8` 或 `10` |
| 明明翻頁了卻沒偵測到 | `CHANGE_THRESHOLD` 太不敏感 | 把數值調低，例如改成 `3` |
| OCR 辨識結果空白或亂碼 | 截到的範圍沒對準文字區塊、或畫面還在翻頁動畫中 | 重新校準座標；或把 `STABILIZE_DELAY` 調高，例如從 `0.6` 改成 `1.0` |
| 翻譯失敗，終端機顯示錯誤 | API Key 沒設定好，或額度用完 | 確認 `GEMINI_API_KEY` 環境變數有生效；到 Google AI Studio 檢查額度 |
| 執行 `pip install` 出現權限錯誤 | 系統權限問題 | 在指令後面加 `--user`，或用系統管理員身分開終端機 |
| 通知沒有跳出來，但筆記檔案有更新 | `win11toast` 在你的 Windows 版本上顯示失敗 | 不影響核心功能可以忽略；或檢查終端機印出的錯誤訊息 |

## 五、之後想備援模型時

目前預設用 `gemini-3.5-flash`。如果額度用完或想省成本，可以把設定區的：

```python
GEMINI_MODEL = "gemini-3.5-flash"
```

改成：

```python
GEMINI_MODEL = "gemini-3.1-flash-lite"
```

`gemini-3.1-flash-lite` 是低延遲、低成本的模型，官方就是拿翻譯場景當主要範例，適合當備援。
