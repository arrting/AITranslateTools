# Kindle 自動翻譯筆記工作流 — 完整使用說明

適用腳本:`fixed_region_watcher.py`(主程式)+ `find_coordinates.py`(座標校準工具)

**這套工具做什麼?**
你在 Windows 上用 Kindle App 讀英文小說,程式會自動偵測你翻頁,把每一頁的文字 OCR 辨識出來、用 Gemini 翻譯成繁體中文,完整譯文即時顯示在終端機,同時依章節寫進筆記檔。你只要專心閱讀、翻頁。

---

## 一、環境建置(只要做一次)

### 步驟 1:安裝 Python

1. 開啟終端機(按 `Win + R` 輸入 `cmd`,或搜尋「PowerShell」),輸入 `python` 按 Enter
2. **如果跳出 Microsoft Store 或 Python 安裝管理器**:照提示安裝
   - 安裝管理器問「Add commands directory to PATH now? [y/N]」→ 輸入 `y`
   - 問「Install CPython now? [Y/n]」→ 直接按 Enter(預設就是安裝)
3. 也可以到 [python.org/downloads](https://www.python.org/downloads/) 下載安裝,記得勾選「Add python.exe to PATH」
4. **裝完後關掉終端機、重新開一個**(PATH 更新後要重開才生效),確認安裝成功:

```powershell
python --version
```

看到類似 `Python 3.14.6` 就成功了。

### 步驟 2:安裝 Python 套件

```powershell
python -m pip install pyautogui winocr pillow google-genai win11toast
```

> 💡 有些電腦 `pip` 指令不在 PATH 裡,用 `python -m pip` 一定可以。
> 如果出現權限錯誤,在指令最後加上 `--user`。

### 步驟 3:安裝 Windows 英文 OCR 語言套件

OCR 用的是 Windows 內建引擎,需要英文辨識套件。**二選一**:

**方法 A:用設定介面(建議,比較不會卡住)**

1. 開啟「設定」→「時間與語言」→「語言與地區」
2. 按「新增語言」→ 搜尋 `English` → 選「**English (United States)**」→ 下一步
3. 「語音辨識」「文字轉語音」「手寫」可以不勾(省空間)
4. ⚠️ **千萬不要勾**「設定為我的 Windows 顯示語言」,不然整個 Windows 介面會變英文
5. 按「安裝」,等進度跑完(可能要幾分鐘,進度條有時看起來不動但其實在下載)

**方法 B:用指令**

對「開始」按右鍵 → 「終端機(系統管理員)」,執行:

```powershell
Add-WindowsCapability -Online -Name "Language.OCR~~~en-US~0.0.1.0"
```

> 💡 如果你開到的是 cmd(會顯示「不是內部或外部命令」),改用這行:
> `DISM /Online /Add-Capability /CapabilityName:Language.OCR~~~en-US~0.0.1.0`
>
> ⚠️ 這個指令有時會卡很久沒進度,卡超過 15 分鐘就 `Ctrl+C` 取消,改用方法 A。

### 步驟 4:申請 Gemini API Key

1. 前往 [Google AI Studio](https://aistudio.google.com/) 登入 Google 帳號,建立 API Key
2. **建議用個人電腦、個人 Gmail 帳號**,公司/機構帳號的瀏覽器政策可能會擋掉申請流程
3. 複製產生的金鑰(一長串英數字)

### 步驟 5:設定環境變數

在終端機(PowerShell 或 cmd 都可以)執行:

```powershell
setx GEMINI_API_KEY "你的API金鑰"
```

⚠️ 設定完**必須關掉終端機重開**,環境變數才會生效。

### 步驟 6:取得腳本

把 `fixed_region_watcher.py` 和 `find_coordinates.py` 放在同一個資料夾(例如 `D:\Codes\AITranslateTools`)。

## 二、閱讀前設定

### 步驟 1:安排視窗位置(很重要!)

因為程式會即時在終端機顯示完整譯文,建議這樣擺:

- **雙螢幕**:Kindle 放一個螢幕,終端機放另一個,最省事
- **單螢幕**:Kindle 視窗靠左佔約 2/3,終端機放右邊,**兩者不能重疊**
  (如果終端機蓋到監控範圍,程式會把終端機的字也截進去,整個亂掉)

擺好之後 Kindle 視窗的大小、位置就**盡量不要再動**,一動就要重新校準座標。

### 步驟 2:校準座標

1. Kindle 開好書、視窗擺定位
2. 執行:

```powershell
cd D:\Codes\AITranslateTools
python find_coordinates.py
```

3. 依終端機提示,5 秒內把滑鼠移到 Kindle **文字內容區塊的左上角**,停住等倒數結束
4. 再依提示把滑鼠移到**右下角**,停住等倒數結束
5. 程式會印出:

```
REGION = (1, 122, 1248, 1026)
```

6. 打開 `fixed_region_watcher.py`,把設定區的 `REGION = (...)` 換成這行的數值

> 💡 框選範圍不用刻意閃避 Kindle 的頁首書名、頁尾頁碼(「Location 533 of 4215 • 12%」那些),程式會自動過濾掉這些雜訊。

### 步驟 3:修改設定區

打開 `fixed_region_watcher.py`,最上面的設定區依你的狀況修改:

```python
REGION = (1, 122, 1248, 1026)   # 上一步校準出來的座標

NOTE_FILE = r"D:\Codes\TEST\HIM.txt"          # 英文原文存這裡
TRANSLATION_FILE = r"D:\Codes\TEST\HIM_翻譯.txt"  # 中文翻譯存這裡
SEPARATE_FILES = True     # True=原文/翻譯分兩個檔;False=合併存在 NOTE_FILE 一個檔
BOOK_TITLE = "HIM"        # 書名(也用來過濾頁首的書名雜訊,要跟畫面上顯示的一致)
CHAPTER = "第五章"          # 起始章節(之後會自動偵測換章+從筆記檔接續,通常只有第一次要填對)
CHAPTER_TITLE = "衛斯(WES)"  # 章節副標題,會顯示在編號下一行;沒有就留空 ""
AUTO_CHAPTER = True        # 自動偵測章節頁並切章(見下方說明);格式特殊的書可改 False 退回手動
NAME_GLOSSARY = {          # 這本書的人名固定譯名(換書時換成新書的角色;不需要可清空成 {})
    "Wes": "衛斯",
    "Jamie": "傑米",
    # ...
}
```

**章節怎麼運作(v2 之後)**:

- 翻到章節頁(如「SIX / JAMIE」「CHAPTER 6」,日文書的「第2章」)時,程式**自動切換章節**,
  標題寫成「第六章 / 傑米(JAMIE)」的格式,副標人名有在譯名表裡就自動帶中文
- 重啟程式時會**自動讀筆記檔的最後一個章節標題接續**,不用改設定
- 所以 `CHAPTER` / `CHAPTER_TITLE` 通常只有「第一次開新書、又不是從章節頁開始讀」時才需要手動填

**人名對照表(NAME_GLOSSARY)**:

- 每頁是獨立翻譯,不給對照表的話,同一個角色的譯名可能每頁不一樣(Wes 變成衛斯/威斯/韋斯...)
- 建議把主要角色都列進去;讀到一半發現新角色譯名飄移,隨時補進表裡即可(重啟生效)

筆記檔和資料夾**不用事先建立**,程式會自動產生。筆記裡的章節標題會長這樣:

```
HIM

第五章
衛斯(WES)

(這一章的內容...)
```

其他設定通常不用動:

```python
OCR_LANG = "en"          # OCR 辨識語言:英文 "en"、日文 "ja"(讀日文書時改這裡,見第六節)
MODEL_CHAIN = [                          # 翻譯模型鏈:第一個是主力,失敗時依序往後退
    "gemini-3.1-pro-preview",            # 主力(Google AI Pro 方案可用,品質好但較慢)
    "gemini-3.5-flash",                  # 第一備援:速度快(已關思考模式)
    "gemini-3.1-flash-lite",             # 第二備援:最省額度
]
POLL_INTERVAL = 1.0      # 每隔幾秒檢查一次畫面
STABILIZE_DELAY = 0.6    # 偵測到翻頁後等動畫穩定的秒數
CHANGE_THRESHOLD = 5     # 畫面變化靈敏度(越小越敏感)
```

## 三、每次閱讀操作

1. 打開 Kindle,放到校準時的位置,翻到你要開始讀的那一頁
2. 確認 `fixed_region_watcher.py` 裡的 `CHAPTER`、`CHAPTER_TITLE` 是目前的章節
3. 開終端機執行:

```powershell
cd D:\Codes\AITranslateTools
python fixed_region_watcher.py
```

4. 程式啟動後會**先翻譯當前畫面上這一頁**(不會漏掉第一頁),之後每翻一頁自動處理
5. 每頁完成後:
   - 終端機顯示**完整譯文**,並標註這一頁用的模型
   - 右下角跳出通知(只是提示,不影響程式繼續運作)
   - 原文和譯文自動寫入筆記檔
6. 讀完想結束,回到終端機按 `Ctrl + C`

### 讀到新章節時

**不用做任何事**——翻到章節頁時程式會自動偵測並切章。只有在 `AUTO_CHAPTER = False` 或章節頁格式認不出來時,才需要手動改 `CHAPTER`/`CHAPTER_TITLE` 重啟。

### 跨 session 閱讀 / 回頭補看

- 筆記依章節分區存放,中斷後接著讀不影響;重啟時程式會自動從筆記檔接續最後的章節
- 之前翻過的頁面重啟後不會重複翻譯(程式會檢查筆記檔已有的內容)
- 如果往回翻頁補看漏掉的內容,程式會把它插進**對應章節的區塊**,不會因為順序打亂筆記

### 換一本新書時的檢查清單

1. `BOOK_TITLE` 改成新書名(照 Kindle 頁首顯示的填)
2. `NOTE_FILE` / `TRANSLATION_FILE` 換一組新檔案路徑
3. `NAME_GLOSSARY` 清空或換成新書的角色譯名
4. `CHAPTER` / `CHAPTER_TITLE` 填你開始讀的章節(如果是從第一章的章節頁開始讀,隨便填也行,自動偵測會蓋過去)
5. 讀日文書多改 `OCR_LANG = "ja"`(見第六節)
6. Kindle 視窗位置變了就重新校準 `REGION`

### 自動容錯機制(不用你操作,知道就好)

- Gemini 過載(503)時:自動等待重試,最多兩次
- 重試仍失敗:沿著模型鏈逐級換備援翻這一頁(`gemini-3.5-flash` → `gemini-3.1-flash-lite`)
- 模型回 404/403(例如 API key 沒有 preview 模型的權限):不重試,直接換下一個模型
- 內容被安全機制擋下:自動切成小段分別翻譯;這個補救階段直接改用 flash 系列(跳過 pro,較快也較不容易再被擋)
- 整條模型鏈都失敗:這一頁會被記住,下一輪自動再試,直到成功,不會漏頁
- 終端機會標示每頁實際用的模型,一看就知道有沒有切到備援

## 四、常見問題排查

| 狀況 | 可能原因 | 解決方式 |
|---|---|---|
| 執行 `python` 沒反應或跳出 Microsoft Store | Python 沒真正安裝 | 回到「環境建置步驟 1」安裝 |
| `pip` 不是內部或外部命令 | pip 不在 PATH | 改用 `python -m pip ...` |
| 錯誤訊息含 `Add-WindowsCapability ... Language.OCR` | 英文 OCR 套件沒裝 | 回到「環境建置步驟 3」 |
| 翻譯失敗:No API key was provided | 環境變數沒生效 | 確認做過 `setx GEMINI_API_KEY`,並**重開終端機** |
| 翻譯失敗:503 UNAVAILABLE / high demand | Google 伺服器尖峰過載 | 不用處理,程式會自動重試+換備援模型 |
| 翻譯失敗:429 / RESOURCE_EXHAUSTED | 免費額度用完 | 到 Google AI Studio 查額度,或等額度重置 |
| 程式沒反應,翻頁也沒觸發 | REGION 座標不準或 Kindle 視窗被移動過 | 重新校準座標;Kindle 位置盡量固定 |
| 一直誤判「翻頁了」(畫面沒變卻觸發) | 靈敏度太高,或終端機/其他視窗蓋到監控範圍 | `CHANGE_THRESHOLD` 調高(如 8~10);確認監控範圍內只有 Kindle |
| 明明翻頁了卻沒偵測到 | 靈敏度太低 | `CHANGE_THRESHOLD` 調低(如 3) |
| OCR 結果空白或亂碼 | 範圍沒對準文字,或截到翻頁動畫 | 重新校準;`STABILIZE_DELAY` 調高(如 1.0) |
| 譯文混入頁碼、書名等雜訊 | 過濾規則沒涵蓋到 | 確認 `BOOK_TITLE` 跟畫面顯示一致;仍有問題可回報調整過濾規則 |
| 譯文沒有分段、一整坨 | 模型偶發 | 提示詞已要求分段,偶爾一頁沒分好屬正常波動 |
| 通知沒跳出來,但筆記有更新 | Windows 通知設定或 win11toast 相容性 | 不影響核心功能,可忽略 |
| 某一頁等很久才出翻譯 | 撞上 Gemini 尖峰,程式在重試 | 終端機會顯示重試進度,等它自己完成即可 |

## 五、模型調整

| 需求 | 做法 |
|---|---|
| 翻譯品質優先(目前預設,需 Google AI Pro 方案) | `MODEL_CHAIN` 第一個放 `"gemini-3.1-pro-preview"` |
| 速度優先 | `MODEL_CHAIN` 第一個放 `"gemini-3.5-flash"` |
| 省額度優先 | `MODEL_CHAIN` 第一個放 `"gemini-3.1-flash-lite"` |

`MODEL_CHAIN` 是照順序退的備援鏈,調整順序或增減模型都可以,程式會從第一個開始用,失敗才往後退。flash 系列預設**關閉思考模式**(翻譯用不到推理),一頁約 1~2 秒;pro 系列不允許關閉思考,一頁會多花幾秒,換來更好的譯文。改完設定記得重啟程式才生效。

## 六、讀日文書

### 一次性準備:安裝日文 OCR 套件

跟安裝英文 OCR 套件的方式一樣,**二選一**:

- **設定介面**:「設定」→「時間與語言」→「語言與地區」→「新增語言」→ 搜尋「**日本語**」→ 安裝(一樣**不要勾**「設定為我的 Windows 顯示語言」)
- **指令**(管理員終端機):

```powershell
Add-WindowsCapability -Online -Name "Language.OCR~~~ja-JP~0.0.1.0"
```

### 每次切換到日文書時,改設定區這幾項

```python
OCR_LANG = "ja"                  # 關鍵:OCR 改用日文辨識
BOOK_TITLE = "書名(照 Kindle 頁首顯示的填)"  # 用來過濾頁首書名雜訊
NOTE_FILE = r"D:\...\日文書.txt"   # 換一組新的筆記檔,別跟英文書混在一起
TRANSLATION_FILE = r"D:\...\日文書_翻譯.txt"
CHAPTER = "第一章"
CHAPTER_TITLE = ""               # 有副標題就填,沒有留空
```

如果 Kindle 視窗位置跟之前不同,記得重跑 `python find_coordinates.py` 更新 `REGION`。

### 已內建的日文支援(不用你動手)

- 日文版 Kindle 的頁尾雜訊(「位置No. 533 / 4215」「残り○分」「ページ」等)會自動過濾
- 日文 OCR 的行與行之間不會插入多餘空格(英文才需要空格接行)
- 翻譯提示詞不限定原文語言,日文一樣翻成繁體中文

### 切回英文書時

把 `OCR_LANG` 改回 `"en"`,並把 `BOOK_TITLE`、筆記路徑、章節換回對應的值即可。
